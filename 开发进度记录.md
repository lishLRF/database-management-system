## 二、Markdown 智能解析模块

### 2.1 架构概览

```
导航栏 → 📄 Markdown解析 → MarkdownWorkbench.vue (独立页面)
    ↓
MarkdownController (SSE + CRUD)
    ├── /upload          上传文件
    ├── /summarize       SSE: AI生成摘要
    ├── /chunk           SSE: 四阶段管线分块(不存库)
    ├── /slice           代码切分预览(纯逻辑)
    ├── /confirm         人工确认后存库
    ├── /feedback        SSE: 人工反馈→AI调整
    ├── /export          导出两文件
    └── /chunks/:id      查询/修改分块
```

### 2.2 分块管线 (Phase 0a → 0b → 1 → 1.5 → 2 → 3)

```
Phase 0a: 代码预扫 (0 AI)
  ├── 硬标题: # / ## markdown标题 — 无条件信任
  ├── 候选标题: 中文编号(一、/1.1) + **粗体行** — 待AI验证
  ├── 表格范围: |...| 连续行
  ├── 代码块范围: ```...```
  └── 段落边界: \n\n 位置

Phase 0b: AI验证候选标题 (0-1次AI批量调用)
  └── 每个候选带上下文(上文80字+下文120字) → AI返回确认编号列表

Phase 1: 原子拆分 (代码 + AI处理超长段)
  ├── 所有结构边界 → 切分为原子片段 (>20字符)
  ├── >3000字符的段 → AI语义拆分
  └── 片段数量不受限制

Phase 1.5: 纯度预检 (代码计算 + 1次AI批量确认)
  ├── 代码计算数据密度 = 数据特征字符 / 总有效字符
  ├── <15%  → 纯语义 → 直接分类
  ├── 15%-75% → 疑似混合 → AI确认需拆分编号
  └── >75% → 纯数据 → 直接分类为structured_data

Phase 2: 工具调用分类 (N次AI调用)
  ├── [TOOL:classify_segment{start,end,type,title,reason}]
  ├── [TOOL:split_segment{start,end,split_at,left_type,right_type,...}]  ← 仅混合段可用
  ├── [TOOL:skip_segment{start,end,reason}]
  └── [TOOL:done{}]

Phase 3: 相邻合并 + 五维验证 (0 AI)
  ├── 相邻同类型合并 (间距<200字符)
  ├── D1: 覆盖完整性 → 自动补gap
  ├── D2: 重叠检测 → 自动调边界
  ├── D3: 边界吸附 → 吸附到\n\n±50字符
  └── D5: 过小块警告 (<20字符)
```

### 2.3 工作流

```
上传 → 摘要 → 2️⃣分块(AI四阶段管线)
                ↓
          右侧代码切片预览(纯代码)
                ↓
         人看策略 + 右侧预览
                ↓
    ├─ 满意 → ✅确认存库 → 两个独立下载按钮
    └─ 不满意 → 写反馈 → 发送 → AI调整策略 → 右侧更新
```

## 三、输出上限统一 (maxTokens)

### 3.1 统一规则

所有 AI 调用点不再硬编码 `4096`：

| 文件 | 调用点 | 公式 |
|------|--------|------|
| `MarkdownAgent.java` | summarize / aiAtomicSplit / classifySegments / feedbackAndRechunk | `outputTokens(config)` |
| `AgentService.java` | Planner / Executor | `outputTokens(config)` |
| `AiService.java` | NL→SQL / 纠错 / 解释 (非流式) | `outputTokens(config)` |
| `AiService.java` | 流式对话 | `config.getMaxTokens()` (全量) |

### 3.2 计算公式

```java
outputTokens(config)  = max(config.maxTokens / 3, 16384)   // = 33333 (设100000时)
toolLoopTokens(config) = min(outputTokens, 16384)           // 工具循环专用
```

### 3.3 推理模型兼容

DeepSeek V4 Pro 等推理模型的 API 返回中 `content` 可能为空，实际输出在 `reasoning_content`。

`callAI()` 自动处理：`content` 为空时 → 从 `reasoning_content` 提取工具调用/JSON/兜底尾部。
